How to Report Issues

If you've found a bug or want to suggest a feature, here's how you can help:


Step 1: Check Existing Issues

Before you create a new issue, please check if someone else has reported the same issue. This avoids duplicates and speeds up resolution.

Step 2: Create a New Issue

If your issue is unique, go ahead and create a new issue on the GitHub repo.

Step 3: Explain the New Issue

Here is an optional template for you to fill out in the "Leave a comment" section


  • Title - Short and descriptive.
  • Description - Detailed explanation of the issue.
  • Steps to Reproduce - Step-by-step guide to reproduce the bug.
  • Expected Behavior - What you expected to happen.
  • Actual Behavior - What actually happened.
  • Screenshots - If applicable.
